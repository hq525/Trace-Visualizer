`npm i source-map-js`; copy `tree-sitter-{javascript,typescript,tsx}.wasm` from `spike/node_modules/tree-sitter-wasms/out/` into `grammars/`. Commit.

### Task 2: pytest-default extraction → corpus 32/32
- New anchor in `src/extract/index.ts` for pytest long format: a `^_{5,} .+ _{5,}$` test-divider followed (within the section) by `E   <Type>: <msg>` lines and a final `^(\S+):(\d+): (\w+)$` location line. Emit a single-frame python trace: `{rawPath, line, symbol: <test name from divider>}`, exception from the location line's type + first `E` line's message.
- Hand-audit + freeze `fixtures/traces/17-py-pytest-default.golden.json` (goldenOrigin noted). Corpus test's gap-assertion flips to expecting a parse. TDD: golden first (test fails), then anchor.

### Task 3: JS/TS tree-sitter analysis (`src/analyze/javascript.ts`)
Symbols: `function_declaration`, `generator_function_declaration`, `method_definition` (class-qualified), `class_declaration`, and `function/arrow_function` bound via `variable_declarator` / `pair` / `assignment_expression` / `public_field_definition`. Imports: `import_statement` named+default+namespace, `require()` calls. Calls: `call_expression` callee identifier / member last segment, with innermost enclosing function. Grammar per extension (`.ts`→typescript, `.tsx`→tsx, else javascript). Test mirrors tests/analyze.test.ts with a TS source containing a class method, arrow const, default-import call.

### Task 4: JS frames through the resolver
- `src/resolve/index.ts`: route `.js/.jsx/.mjs/.cjs/.ts/.tsx` files to the JS analyzer (extract `analyzeFile` dispatch); normalize trace symbols (`strip async/new`, `[as alias]`, dotted last-segment — port `normSymbol` from spike/resolve.mjs); anonymous frames (no symbol) resolve to the innermost containing function by span (no name check, no badge).
- Test: temp repo with a TS file; frames for a method (`Ledger.append` printed form), an arrow (anonymous, bare loc), `async fn`, plus a drifted line → badge.

### Task 5: Sourcemap stage (`src/sourcemap/index.ts`, §5.3)
```ts
export async function applySourcemaps(trace: ParsedTrace, index: RepoIndex): Promise<ParsedTrace>
```
For each non-external frame with `column` whose matched repo file is generated (path contains `dist/`, `build/`, `.next/`, or file content contains `sourceMappingURL`): locate map (inline data-URL → adjacent `<file>.map` → sourceMappingURL relative path), `SourceMapConsumer.originalPositionFor({line, column})`, rewrite rawPath/line/column/symbol(name??keep), set `mappedFrom` + implicit badge `via-sourcemap` (carried on Frame, surfaced by resolver into node badges). Broken/missing → `no-sourcemap` badge, frame untouched. Wire into `runPipeline` before `resolveTrace`. Frame type gains `mappedFrom?: string`. Test: temp repo assembled from `fixtures/traces/27-js-minified-sourcemap.bundle.js{,.map}` + `spike/gen/js/src/*.ts` as `src/`; run pipeline on fixture 27's trace text; assert crash frame resolves into `src/pricing.ts` with badges `via-sourcemap`.

### Task 7: Multi-trace picker (§5.1.4)

- Server: `POST /api/trace` → when >1 distinct trace and no `pick`, `200 {picker: [...]}`; with `pick: i` builds that graph. GET /api/graph unchanged.
- UI: picker screen (list of signatures → onPick posts again with pick). Tests: blob = fixtures 01+21 concatenated → picker with 2 entries; pick=1 → js graph.
